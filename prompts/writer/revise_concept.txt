你要帮助用户修改小说构思。

修改范围：{scope}
目标对象：{target_id}
用户要求：{guidance}

当前小说 JSON：
{book_json}

参考写作技巧与反模式卡：
{reference_pack}

输出要求：
1. 如果 scope = all，输出 JSON：
{{
  "title": "...",
  "premise": {{...}},
  "characters": [{{...}}],
  "chapter_plans": [{{...}}]
}}
注意：scope = all 只能修改“当前小说 JSON 中已经存在的部分”。如果当前 `characters` 是空数组，必须继续返回空数组，不能新增角色；如果当前 `metadata.chapter_plans` 是空数组，必须继续返回空数组，不能新增章节规划。生成角色和章节规划只能由独立任务完成。
2. 如果 scope = premise，输出 JSON：
{{
  "premise": {{...}}
}}
3. 如果 scope = character，输出 JSON：
{{
  "character": {{...}}
}}
4. 如果 scope = chapter_plan，输出 JSON：
{{
  "chapter_plan": {{...}}
}}
5. 必须保持字段结构合法，不能缺字段。
6. 修改要尊重用户要求，保留已有世界观和已写正文能兼容的部分。
7. 如果修改涉及 `title`，标题必须有画面、记忆点和题材气味，能暗示核心困局或人物命运；避免“XX逆袭”“XX觉醒”“命运之XX”“重生之XX”这类泛标题。必要时可用“主标题：副标题”，但主标题必须干净有辨识度。
8. 如果修改涉及 premise，不要只强化“女性觉醒/成长/救赎/权谋博弈”等主题词，要把构思压成硬故事问题：主角做过什么不可轻易原谅的选择，为什么当时非做不可，现在这个选择如何回来索债。
9. 主角不能只是“被体系伤害的无辜者”。要保留或补足她的原罪、私心、误判或代价；后续成长应来自承认选择、承担后果，而不是被洗白。
10. 如果参考卡片指出人物、关系、情节推进或反模式问题，要把它当作写作技巧约束使用；不要机械复述卡片标题。
11. 修改角色和章节规划时，优先补足人物目标、恐惧、底线、行为逻辑、关系旧账和具体行动因果，避免人物没立起来就推进剧情。
12. 只输出 JSON，不要解释。
