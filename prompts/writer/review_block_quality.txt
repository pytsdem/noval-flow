Check only these things:
1. Check whether the block completes the stated `scene_goal` and reaches the `end_state`.
2. Check whether the block feels like a scene unit instead of a summary, outline, or bridge paragraph.
3. Check whether it realizes the `human_reaction_target` with something specifically human on the page.
4. Check whether the `cost_shift` actually lands.
5. Check whether the intended `reader_feeling_target` can be felt from the block.
6. Check whether any paragraph is over 180 Chinese characters.
7. Check whether any paragraph carries too many functions at once.
8. Check whether dialogue that should stand alone is buried inside a dense paragraph.
9. Check whether an important side-character reaction that should stand alone is buried inside a dense paragraph.
10. Check whether the block violates chapter-internal continuity and reads like a restart instead of a continuation.
11. Check whether the block violates `style_risk_guard`.
12. Check whether emotion is too directly stated instead of externalized through action, pause, body, object, or consequence.
13. Check whether the block contains a character anchor line.
14. Check whether the anchor line is merely functional instead of also revealing character.
15. Check whether the anchor line is immediately over-explained by following narration.
16. Check whether the anchor line is memorable enough to help the character stand up on the page.
17. If `clue_reveal_mechanism` exists, check whether the clue appears naturally rather than as an author-arranged drop.
18. If `character_reentry_mode` exists, check whether the re-entry feels natural instead of re-explaining identity.

Required issue categories:
- `paragraph_too_long`
- `paragraph_function_overloaded`
- `dialogue_not_separated`
- `side_reaction_not_separated`
- `continuity_restart`
- `missing_human_reaction_target`
- `missing_cost_shift`
- `weak_reader_feeling_target`
- `missing_character_anchor_line`
- `anchor_line_too_functional`
- `anchor_line_overexplained`

Optional categories when needed:
- `scene_goal`
- `outline_prose`
- `style_risk_violation`

Trigger rule:
- If one paragraph contains 3 or more of these functions at once, flag `paragraph_function_overloaded`:
  `main action / inner reaction / side-character reaction / explanation / world-rule reminder / clue / object emphasis`

Evidence rules:
1. Never say only "flat", "weak", "not human enough", or "hook is weak".
2. Every issue must include:
   - `evidence`: a concrete beat, paragraph problem, or an important missing beat
   - `reason`: why that evidence hurts the block
   - `fix`: how to fix this block without rewriting the whole chapter
3. Prefer 0 to 4 issues. Keep the review light and local.
4. Evidence should mention paragraph position or concrete content whenever possible.
5. Anchor-line fixes should prefer:
   - 增加一句短而锋利的角色锚句
   - 把功能句改成“表层功能 + 深层揭示人物”的句子
   - 删掉锚句后立刻跟着的解释性旁白
   - 用配角反应或停顿来托住锚句
