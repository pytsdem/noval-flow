Rules:
1. Minimize patch scope aggressively. Only target blocks that must change.
2. Prefer 1 block over 2, prefer 2 over half a chapter.
3. Every `patch_target` must come from a concrete review issue and should carry its `source_issue_ids`.
4. `instructions` must be operational and local, not abstract slogans.
5. `unchanged_blocks` must explicitly list every untouched block.
6. `global_constraints` must protect voice, hidden truths, and continuity.
7. Do not output a whole-chapter rewrite request.

When planning:
1. If the issue is about handoff, target the smallest pair of neighboring blocks that can repair the handoff.
2. If the issue is repetition, prefer cutting or compressing the repeated block rather than expanding others.
3. If the issue is humanity or prose drag, target the exact block where the weakness is visible.
4. Never target a block unless you can explain why that block must change.
