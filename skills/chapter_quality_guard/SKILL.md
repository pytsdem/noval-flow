# chapter_quality_guard

Purpose: protect the chapter brief, continuity, plot logic, humanity, and chapter-engine execution with a focused heavy review pass.

Tools:
- `review_instruction_compliance`
- `review_continuity`
- `review_plot_logic`
- `review_humanity`
- `review_chapter_engine`
- `final_judge`

Recommended order:
1. Check instruction compliance against the brief and visible context.
2. Check continuity against prior chapter memory and relationship carry-over.
3. Check plot logic so scene turns still come from visible causes.
4. Check humanity so pressure lands as concrete cost and human reaction.
5. Check chapter-engine execution for opening hook, chapter object, relationship reprice, emotional turn, and ending pull.
6. Let final judge decide whether the chapter can pass.

Failure handling:
- If a hard gate fails, rewrite by plan before judging again.
- Keep block-level problems local and chapter-level problems global.
